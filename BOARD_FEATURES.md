# Board Features Analysis - Trello + Jira Combined

## üéØ Core Board Features

### Trello Core
- **Boards** - Visual project workspace
- **Lists/Columns** - Vertical swim lanes (Backlog, To Do, In Progress, Review, Done)
- **Cards** - Individual task items
- **Drag & Drop** - Move cards between lists
- **Card Covers** - Visual headers with colors/images
- **Board Backgrounds** - Custom themes and images
- **Workspaces** - Organize multiple boards
- **Templates** - Pre-built board layouts

### Jira Core
- **Projects** - Container for issues
- **Issues** - Granular task items (Story, Task, Bug, Epic, Subtask)
- **Kanban Boards** - Continuous flow
- **Scrum Boards** - Sprint-based workflow
- **Backlog** - Unprioritized work queue
- **Sprints** - Time-boxed iterations
- **Board Views** - Multiple visualization modes
- **Swimlanes** - Horizontal grouping (by assignee, priority, epic)
- **Quick Filters** - Toggle visibility of card types
- **WIP Limits** - Restrict cards per column

## üìù Card/Issue Management

### Trello Cards
- **Title** - Card name
- **Description** - Rich text details
- **Labels** - Color-coded tags
- **Members** - Assigned users (multiple)
- **Due Date** - Deadline with time
- **Checklists** - Sub-tasks with progress
- **Attachments** - Files, links, images
- **Comments** - Discussion thread
- **Activity Log** - Change history
- **Card Aging** - Visual fading for old cards
- **Card Copying** - Duplicate cards
- **Card Moving** - Cross-board transfer
- **Card Archiving** - Soft delete
- **Voting** - Member voting on cards
- **Custom Fields** - Extra metadata (text, number, date, dropdown)

### Jira Issues
- **Issue Key** - Unique ID (e.g., PROJ-123)
- **Issue Type** - Story, Task, Bug, Epic, Subtask
- **Summary** - Issue title
- **Description** - Rich text with formatting, tables, code blocks
- **Priority** - Highest, High, Medium, Low, Lowest
- **Status** - Workflow state (To Do, In Progress, Done, etc.)
- **Assignee** - Single user assigned
- **Reporter** - User who created issue
- **Labels** - Multiple text tags
- **Story Points** - Effort estimation
- **Time Tracking** - Original estimate, remaining, logged time
- **Components** - Functional areas
- **Versions/Fix Versions** - Release targeting
- **Epic Link** - Parent epic relationship
- **Issue Links** - Relations (blocks, is blocked by, relates to, duplicates)
- **Subtasks** - Child issues
- **Attachments** - Files and screenshots
- **Comments** - @mentions, rich text
- **Watchers** - Users following the issue
- **Transitions** - Workflow actions
- **History** - Full audit trail
- **Cloning** - Duplicate issues

## ü§î Key Design Decisions Needed

1. **Issue Types**: Should we use Jira's (Story/Task/Bug/Epic) or keep simpler?
2. **Assignment**: Single assignee (Jira) or multiple (Trello)?
3. **Estimation**: Story points, time estimates, or both?
4. **Sprints**: Do we need sprint planning or just continuous kanban?
5. **Subtasks**: Inline checklists (Trello) or separate cards (Jira)?
6. **Workflows**: Fixed columns or customizable workflows?
7. **Views**: Just board view or also list/calendar/timeline?
8. **Permissions**: Simple (public/private) or complex roles?

## üìê Proposed Hybrid Model

### **Card Structure**
```
Card {
  id: string
  key: string (e.g., "CHAT-42") - Jira style
  title: string
  description: string (rich text)
  type: 'story' | 'task' | 'bug' | 'epic' - Jira
  priority: 'highest' | 'high' | 'medium' | 'low' | 'lowest' - Jira
  status: string (column name)
  labels: Label[] - Trello style (color + name)
  assignees: User[] - Trello style (multiple)
  reporter: User - Jira
  storyPoints?: number - Jira
  dueDate?: Date - Trello
  checklist: ChecklistItem[] - Trello
  attachments: Attachment[] - Both
  comments: Comment[] - Both
  watchers: User[] - Jira
  epicLink?: string - Jira
  subtasks: string[] - Jira
  timeTracking?: { estimated: number, logged: number } - Jira
  createdAt: Date
  updatedAt: Date
  activityLog: Activity[] - Both
}
```

### **Board Features**
- Kanban columns (drag-and-drop)
- Swimlanes (optional grouping)
- Quick filters (type, priority, assignee, label)
- Search bar
- WIP limits (optional per column)
- Backlog view toggle
- Board settings (background, columns)

### **Views**
1. **Board View** (default) - Kanban
2. **List View** - Table/spreadsheet
3. **Backlog View** - Sprint planning
4. **Calendar View** - Due dates
5. **Timeline View** - Gantt-style (future)
